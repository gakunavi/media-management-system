# cowork 依頼: 記事内のLINEリンクに記事IDを入れる（U72・2026-07-24）

そのまま cowork に貼って使うプロンプト。**MMS 側の対応は完了済みです。**

---

## 依頼の内容（1行）

記事内の公式LINEリンクを `/r/line/{設置場所}` から **`/r/line/{設置場所}-{記事ID}`** に変える。

```
変更前: https://collect.asset-support.co.jp/r/line/media-article-bottom
変更後: https://collect.asset-support.co.jp/r/line/media-article-bottom-ART-159
                                                                    ↑ 記事IDを足すだけ
```

## なぜやるか

いま「どの設置場所が押されたか」は分かりますが、**「どの記事から送られたか」が分かりません**。

記事別の内訳は計測タグ（JS）側でも取れるようになりましたが、**JSが動いた分しか数えられません**（広告ブロック・JS無効・読み込み前の離脱で落ちます）。一方リダイレクタは**サーバー側の実測**なので取りこぼしません。

記事IDを URL に入れると、**サーバー実測だけで「合計」と「記事別の内訳」の両方**が出せます。そうなると「この記事はLINE登録にどれだけ効いたか」が、ブロッカーの影響を受けない数字で言えます。

★現状の実測（2026-07-24・計測タグ側）

| 設置場所 | クリック＋表示 | 関与した記事数 |
|---|---:|---:|
| `site-header` | 22 | 13 |
| `article-cta` | 19 | 15 |
| `header-strip-material` | 12 | 8 |
| `header-strip-question` | 10 | 7 |
| `media-header` | 7 | 5 |

記事数は出ていますが、**どの記事が何件かはサーバー側では取れていません**。

## 仕様（守っていただきたい点）

### 1. 記事IDは末尾に、ハイフン区切りで

```
/r/line/{設置場所}-{記事ID}
```

- 記事IDは **`ART-` ＋ 数字**（例 `ART-159`）。MMS の記事IDと同じもの
- 設置場所（`media-article-bottom` など）は**今の名前のまま変えない**でください
  - 変えると、これまでの設置場所別の実績と繋がらなくなります

### 2. 全体で40文字以内

MMS 側は送り元を `[A-Za-z0-9_-]{1,40}` で受けています。
`media-article-bottom-ART-159` は28文字なので余裕がありますが、
**新しく長い設置場所名を作るときだけ**気にしてください。

### 3. 対象は「記事ページに置かれたリンク」だけ

| 対象 | 記事IDを付ける？ |
|---|---|
| 記事本文・記事末のCTA（`article-cta` / `media-article-bottom` 等） | **付ける** |
| 記事ページのヘッダ・サイドバー・フッタ | **付ける**（その記事を読んでいた人なので） |
| トップ・カテゴリ・固定ページのリンク | **付けない**（記事に紐づかないため） |
| Threads の投稿リンク（`THR-xxx`） | **触らない**（既に投稿IDが入っています） |

### 4. 段階的で構いません

**全記事を一度に変えなくて大丈夫です。** 付いているものだけ記事別に積まれ、
付いていないものは今までどおり設置場所別に残ります。混在しても壊れません。

効果の大きい順に、
① 記事末CTA（`article-cta` / `media-article-bottom`）→ ② ヘッダ帯 → ③ その他
の順で進めていただけると、早く効果が見えます。

---

## MMS 側の対応（完了済み・2026-07-24）

そちらの作業が終われば**その日から記事別に貯まります**。

- 送り元の末尾から `ART-\d+` を取り出し、**記事単位で `ContentMetric` に積む**
- 設置場所の内訳（`site_link_clicks_line__media-article-bottom`）には
  **記事IDを混ぜない**。混ぜると指標が「記事数 × 設置場所」ぶん増えて、
  データの鮮度チェックが壊れるため
- サイト全体の合計はこれまでどおり。**過去の数字は変わりません**
- 存在しない記事IDが来ても、送客そのものは落とさず合計には入れます
  （貼り間違いでクリックを失わないため）

★動作確認済み: `/r/line/media-article-bottom-ART-002` を実際に叩き、
ART-002 に記事別で1件記録され、設置場所の内訳に記事IDが混ざらないことを確認しました。

## 確認したいこと（cowork にお聞きしたい）

1. **記事IDの差し込みは自動でできますか？**
   テーマ側で「現在の記事のMMS記事ID」を取れるなら、テンプレートで
   `/r/line/media-article-bottom-<?php echo $mms_id; ?>` のように
   一括で入れられるはずです。1本ずつ手で貼るなら段階的にお願いします。

2. **記事IDをテーマ側で持っていますか？**
   持っていない場合、MMS から「URL → 記事ID」の対応表（159行のCSV）を
   お渡しできます。必要でしたらお申し付けください。

3. **固定バー（`category-fixed-bar`）は記事ページにも出ますか？**
   出るなら記事IDを付ける対象です。カテゴリページ専用なら付けません。
