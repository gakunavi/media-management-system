## 実装したもの

日次ジョブ **`tag-delivery-daily`（毎日 05:50）** を追加しました。

1. 全公開記事を**読者とまったく同じ条件**で叩く（ブラウザのUA＋`Accept: text/html`・クエリを足さない）
2. 計測タグが入っていない記事を検出する
3. **トークンがあれば、その記事のURLだけをパージする**（Purge Everything は使わない）
4. 結果を `DataQualityCheck(kind=tag_delivery)` に残し、通知は既存の
   `health-alert-daily` がまとめて出す（同じ異常が2箇所から届かないように）

★**すでに実行して、上の6本を独立に再現しています**（151/157 配信あり）。
いまはトークンが無いので、検出だけして止まっています。

## トークンの発行手順（Cloudflare）

### STEP 1: 発行画面を開く

1. Cloudflare にログイン
2. **右上のユーザーアイコン** をクリック → メニューの **My Profile**
3. 左メニューの **API Tokens**
4. **Create Token**（青いボタン）
5. テンプレートが何個か並んだ画面が出ます。**どれも使いません。**
   一番下までスクロールして **Create Custom Token** の右にある
   **Get started** をクリック

### STEP 2: 名前を入れる

一番上の **Token name** に、こう入れます。

```
mms-cache-purge
```

（名前は何でもよく、後から自分が分かればOKです）

### STEP 3: Permissions ← ここが分かりにくい所です

**「Permissions」の欄には、横に3つのプルダウンが並んでいます。**
1つを選ぶのではなく、**左から順に3つとも選びます。**

```
Permissions

┌──────────────┐  ┌──────────────┐  ┌──────────────┐
│  1つ目       │  │  2つ目       │  │  3つ目       │
│              │  │              │  │              │
│   Zone    ▼  │  │ Cache Purge▼ │  │   Purge   ▼  │
└──────────────┘  └──────────────┘  └──────────────┘
      ↑                  ↑                  ↑
  まずここ          次にここ           最後にここ

                                    ＋ Add more  ← 押さない
```

| どのプルダウン | 選ぶもの | 補足 |
|---|---|---|
| **1つ目**（左） | **`Zone`** | 最初は `Account` になっています。`Zone` に変えてください |
| **2つ目**（真ん中） | **`Cache Purge`** | 1つ目を `Zone` にすると選べるようになります。数が多いので `cache` と打つと絞り込めます |
| **3つ目**（右） | **`Purge`** | `Purge` が無ければ **`Edit`** を選んでください（表示はCloudflareの版で違います） |

> **★「この1つだけ」と書いていたのは、「行を1つしか作らない」という意味でした。**
> 3つのプルダウンで**1行**です。その下の **`+ Add more`** は押さないでください。
> 権限を足すほど、万一トークンが漏れたときにできることが増えます。
> DNS・WAF・SSL などの権限は一切必要ありません。

### STEP 4: Zone Resources ← ここも3つ並んでいます

**Permissions の少し下**に「Zone Resources」という欄があります。
ここも横に3つ並んでいるので、左から順に選びます。

```
Zone Resources

┌──────────────┐  ┌──────────────┐  ┌────────────────────────┐
│  Include  ▼  │  │Specific zone▼│  │ asset-support.co.jp  ▼ │
└──────────────┘  └──────────────┘  └────────────────────────┘
```

| どのプルダウン | 選ぶもの |
|---|---|
| **1つ目** | **`Include`**（最初からこうなっているはずです） |
| **2つ目** | **`Specific zone`** |
| **3つ目** | **`asset-support.co.jp`** |

> ★ここで `All zones` を選ばないでください。**このドメインだけ**に限定します。

### STEP 5: 残りは触らない

下の方に **Client IP Address Filtering** と **TTL** がありますが、
**どちらも空のままで構いません**（TTL を空にすると無期限になります）。

### STEP 6: 発行する

1. 一番下の **Continue to summary**
2. 確認画面にこう出ていればOKです

   > `asset-support.co.jp - Cache Purge:Purge`

   ★もし他の行も出ていたら、**Edit** で戻って余計な行を消してください

3. **Create Token**
4. **長い文字列が表示されます。これをコピーしてください**

> ★**この文字列は、この画面を閉じると二度と表示されません。**
> 閉じてしまったら、そのトークンは削除して発行し直すことになります。
> 先に下の「私に渡していただく形」を開いておいて、すぐ貼れる状態にしておくと安全です。

## Zone ID の取得（トークンとは別物・こちらも必要です）

1. Cloudflare ダッシュボードの**ホーム**に戻る

## 私に渡していただく形

`.env` に**石井さんご自身で**追記してください（★値を私に貼る必要はありません）。

`.env` は、このフォルダの直下にあります。

```
~/システム開発/Next/media-management-system/.env
```

一番下に、この2行を足してください。

```
MMS_CLOUDFLARE_API_TOKEN=ここにコピーしたトークンを貼る
MMS_CLOUDFLARE_ZONE_ID=ここにZone IDを貼る
```

> ★`=` の前後に**スペースを入れない**でください。
> ★値を `"` で囲む必要はありません。

追記して保存したら、これを実行すると反映されます。

```bash
cd ~/システム開発/Next/media-management-system && docker compose up -d worker
```

★`.env` は git にコミットされません（§12-4）。`.env.example` には**キー名だけ**追加済みです。

## うまくいかないときの見分け方

私の方で実行すると、こう出ます。

| 出方 | 意味 | 直し方 |
|---|---|---|
| `パージ: 成功 — N件をパージした` | ✅ 完了 | — |
| `MMS_CLOUDFLARE_API_TOKEN / MMS_CLOUDFLARE_ZONE_ID が未設定` | `.env` が worker に渡っていない | `docker compose up -d worker` を実行したか確認 |
| `HTTP 400` | **Zone ID が違う**（別ドメインのIDを貼った等） | Zone ID を取り直す |
| `HTTP 403` | **権限が足りない** | STEP 3 の3つ目が `Purge`（または `Edit`）になっているか確認 |
| `success=false` | 権限はあるが対象ドメインが違う | STEP 4 で `asset-support.co.jp` を選んだか確認 |

★どれが出たかを教えていただければ、こちらで切り分けます。
