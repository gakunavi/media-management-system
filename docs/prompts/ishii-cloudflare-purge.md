## 実装したもの

日次ジョブ **`tag-delivery-daily`（毎日 05:50）** を追加しました。

1. 全公開記事を**読者とまったく同じ条件**で叩く（ブラウザのUA＋`Accept: text/html`・クエリを足さない）

## トークンの発行手順（Cloudflare）

> ★画面は**日本語表示**です（英語表示の場合は括弧内の語）。

### STEP 1: 発行画面を開く

1. Cloudflare にログイン
2. **右上のユーザーアイコン** → **マイプロフィール**（My Profile）
3. 左メニューの **API トークン**（API Tokens）
4. **トークンを作成する**（Create Token）
5. テンプレートが並んだ画面が出ます。**どれも使いません。**
   一番下までスクロールして **カスタム トークンを作成する**（Create Custom Token）
   の **利用を開始する**（Get started）

### STEP 2: トークン名

一番上の **トークン名** に入れます（名前は何でも構いません）。

```
mms-cache-purge
```

### STEP 3: 権限（Permissions）

**「権限」の欄は、横に3つのプルダウンが並んで“1行”です。**
どれか1つを選ぶのではなく、**左から順に3つとも**選びます。

```
権限

┌──────────────┐  ┌────────────────┐  ┌──────────────┐
│   ゾーン  ▼  │  │キャッシュパージ▼│  │  パージ   ▼  │
└──────────────┘  └────────────────┘  └──────────────┘
      ↑                   ↑                   ↑
    1つ目               2つ目               3つ目

  ＋ さらに追加する  ← 押さない
```

| プルダウン | 選ぶもの（日本語表示） | 英語表示 |
|---|---|---|
| **1つ目**（左） | **`ゾーン`** ※最初は「アカウント」 | `Zone` |
| **2つ目**（中） | **`キャッシュ パージ`** ※`cache` と打つと絞れます | `Cache Purge` |
| **3つ目**（右） | **`パージ`** ※無ければ `編集` | `Purge` / `Edit` |

> **★「この1つだけ」と書いていたのは「行を1つしか作らない」という意味でした。**
> 3つのプルダウンで**1行**です。下の **`＋ さらに追加する`** は押さないでください。
> 権限を足すほど、万一トークンが漏れたときにできることが増えます。
> DNS・WAF・SSL などの権限は一切必要ありません。

### STEP 4: ゾーン リソース（Zone Resources）← ★ここを間違えやすい

権限の少し下にあります。**最初は「すべてのゾーン」になっているので変えます。**

```
ゾーン リソース

  変更前 ❌                          変更後 ✅
┌────────┐ ┌──────────────┐     ┌────────┐ ┌────────────┐ ┌────────────────────┐
│ 含む ▼ │ │すべてのゾーン▼│ →  │ 含む ▼ │ │特定のゾーン▼│ │asset-support.co.jp▼│
└────────┘ └──────────────┘     └────────┘ └────────────┘ └────────────────────┘
                  ↑                              ↑                    ↑
             ここを変える                  これを選ぶと      3つ目が現れるので選ぶ
```

| プルダウン | 選ぶもの（日本語表示） | 英語表示 |
|---|---|---|
| **1つ目** | **`含む`** ※最初からこうなっています | `Include` |
| **2つ目** | **`特定のゾーン`** ※既定の「すべてのゾーン」から変える | `Specific zone` |
| **3つ目** | **`asset-support.co.jp`** ※2つ目を変えると**現れます** | 同じ |

> ★**`すべてのゾーン` のままでも動きます**が、**他のドメインのキャッシュまで
> 消せるトークン**になります。必要なのは1つだけなので必ず絞ってください。

### STEP 5: 残りは触らない

- **クライアント IP アドレス フィルタリング** → **空のまま**
- **TTL**（Start Date → End Date） → **空のまま**（無期限になります）

### STEP 6: 発行する

1. 一番下の **概要に進む**（Continue to summary）
2. 確認画面にこう出ていればOKです

   > `asset-support.co.jp - キャッシュ パージ:パージ`

   ★他の行も出ていたら、**編集**で戻って余計な行を消してください

3. **トークンを作成する**（Create Token）
4. **長い文字列が表示されます。これをコピーしてください**

> ★**この文字列は、この画面を閉じると二度と表示されません。**
> 閉じてしまったら、そのトークンは削除して発行し直すことになります。
> 先に下の「私に渡していただく形」を開いておいて、すぐ貼れる状態にしておくと安全です。

## 私に渡していただく形

`.env` に**石井さんご自身で**追記してください（★値を私に貼る必要はありません）。

`.env` は、このフォルダの直下にあります。

```
~/システム開発/Next/media-management-system/.env
```

一番下に、この2行を足してください。

```
MMS_CLOUDFLARE_API_TOKEN=ここにコピーしたトークンを貼る
MMS_CLOUDFLARE_ZONE_ID=ここにZone IDを貼る
```

> ★`=` の前後に**スペースを入れない**でください。
> ★値を `"` で囲む必要はありません。

追記して保存したら、これを実行すると反映されます。

```bash
cd ~/システム開発/Next/media-management-system && docker compose up -d worker
```

★`.env` は git にコミットされません（§12-4）。`.env.example` には**キー名だけ**追加済みです。
