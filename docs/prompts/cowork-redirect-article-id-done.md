### ★④の検証でこちらのバグが出たので直しました

```
送った source: article-cta-ART-002-ART-002
  修正前: 設置場所 = article-cta-ART-002 ／ 記事 = ART-002   ← NG
  修正後: 設置場所 = article-cta          ／ 記事 = ART-002   ← OK
```

末尾のARTを**1つしか剥がしていません**でした。
そちらの設計では二重付与は起きませんが、**万一残ると設置場所の指標が
記事数ぶん増えてしまう**（記事IDを指標名に混ぜない理由がまさにそれ）ので、
**末尾のARTを全部剥がす**方式に変えて吸収しました。

### 桁数可変・その他の境界ケースも確認しました

```
  article-cta-ART-2                 → 設置場所 article-cta            / 記事 ART-2
  media-header-material-ART-18900   → 設置場所 media-header-material  / 記事 ART-18900
  media-header-ART-002-ART-088      → 設置場所 media-header           / 記事 ART-088（末尾を採用）
  article-cta-art-002               → 設置場所 article-cta            / 記事 ART-002（小文字も可）
  article-cta                       → 設置場所 article-cta            / 記事 なし
```

ご指摘のとおり `ART-\d+` で受けており、3桁固定にはしていません。
**段階移行での ART 有り/無しの混在も問題ありません**（③のとおり）。

## これで何が変わったか

これまで「どの設置場所が押されたか」までしか分かりませんでしたが、
**「どの記事から送られたか」がサーバー実測で出せる**ようになりました。

計測タグ（JS）側でも記事別は取れますが、あちらは広告ブロック・JS無効・
読み込み前の離脱で落ちます。**リダイレクタはサーバー側なので取りこぼしません。**

実クリックが入り次第、記事別の内訳が貯まり始めます
（検証で作ったクリックは実データを汚さないよう削除済みなので、いまは0件です）。

**ありがとうございました。U72 はこれで完了です。**

## 別件で1つだけ、ご判断いただきたいこと（急ぎません）

前回ご指摘した **JSON-LD の `sameAs` にリダイレクタURLが入っている**件です。

```json
"sameAs": [ "https://collect.asset-support.co.jp/r/line/article-cta" ]
```

`sameAs` は本来「同一主体を指す**公式プロフィール**」（公式SNSアカウント等）を
入れる項目なので、送客用リダイレクタを入れるのは用途が違います。
今回のフィルタは href 限定なので**触っていません**（それは意図どおりです）。

「必要なら出所を特定してお直しします」とのことでしたので、
**直すかどうかはそちらのSEO判断にお任せします。**
こちらから見えているのは「Organization スキーマの sameAs に入っている」ことと、
記事ページで2箇所出力されていることまでです。
