# cowork 依頼: HP・記事の `lin.ee` を計測できるリンクに張り替える（2026-07-23）

そのまま cowork に貼って使うプロンプト。

---

## 状況（MMS 側で確認した事実）

公式LINEの友だち数を Messaging API から取れるようにしました。**友だちは7人**です。

一方で「**その7人がどこから来たのか**」が1人も分かりません。MMS の /line 画面はこう出ています。

| 入口 | 状態 | 理由 |
|---|---|---|
| HP | **—(未計装)** | テーマの `lin.ee` が生リンク（`/contact/` 7箇所） |
| メディア（記事） | **—(未計装)** | 記事内の `lin.ee` が生リンク（`/media/` 9箇所） |
| Threads | —(未計装) | 投稿にLINEリンクが無い（22本が投稿待ち・対応済み） |
| 不明 | —(測定不能) | LINE の follow イベントに経路情報が入らない（LINE 仕様） |

★「未計装」は**送っていない**のではなく**測っていない**という意味です。
HP・記事からは送っているはずですが、生リンクなので踏まれても記録が残りません。

MMS 側のリダイレクタは稼働しています。**リンクを張り替えるだけで、その日から測れます。**

## 依頼: `lin.ee` の直リンクを `/r/line/{送り元}` に置き換える

```
変更前: https://lin.ee/5NVLBXA
変更後: https://collect.asset-support.co.jp/r/line/{送り元}
```

`{送り元}` は**設置場所を表す識別子**です。MMS はこれを見て入口を分類します。

| 接頭辞 | 分類される入口 | 使う場所 |
|---|---|---|
| `hp-` で始まる | **HP** | `/contact/`・トップ・固定ページ・ヘッダ/フッタ |
| `media-` で始まる | **メディア（記事）** | `/media/` 配下の記事内CTA |

### 命名の規則（重要）

- **英数字・ハイフン・アンダースコアのみ**、40文字以内（それ以外は記録が捨てられます）
- 設置場所が分かる名前にしてください。**この名前がそのまま画面の内訳になります**

例:

```
https://collect.asset-support.co.jp/r/line/hp-header
https://collect.asset-support.co.jp/r/line/hp-footer
https://collect.asset-support.co.jp/r/line/hp-contact-cta
https://collect.asset-support.co.jp/r/line/media-article-bottom
https://collect.asset-support.co.jp/r/line/media-sidebar
```

同じ場所は同じ名前で統一してください（記事ごとに変えなくて構いません。
記事単位の分解は現状しません）。

### 書かないでほしいもの

- **UTM パラメータ**（`?utm_source=` 等）… MMS が自動で付けます。手で付けると二重になり、GA4 の流入元集計が汚れます
- **`lin.ee` のURL自体**… 遷移先は MMS の環境変数で固定しています。リンク先を変えるときは MMS 側の1箇所を直すだけで全箇所に効きます

## MMS 側でやっていること（確認用）

- Threads のプレビュー生成などの **bot は除外**（`facebookexternalhit` 等）。踏んだ数が水増しされません
- **初回クリックで計測開始を自動記録**（§3）。それまでは「0件」ではなく「未計装」と表示されます
- 遷移先は環境変数で固定しており、**URL を引数で受け取る作りではありません**（オープンリダイレクトになりません）

## 計測の限界（先に共有）

**どのリンクから登録したかは分かりません。** LINE の follow イベントに経路情報が
入らない仕様のためで、これは直せません。取れるのは「**LINEへ何人送ったか**」までです。

そのうえで、いま友だち7人という母数が分かっているので、
「HPから何クリック → 友だちが何人増えたか」の対応は追えるようになります。

## 確認方法

1. 張り替え後、実際にリンクを1回踏む
2. MMS の `/line` を開く →「入口」の HP または メディアが `—(未計装)` から実数に変わる
3. 内訳（`hp-header` / `media-article-bottom` など）は設置場所別に積み上がります

作業後に「どの場所に何という名前を付けたか」の一覧を返してください。
MMS 側の分類（HP / メディア）が意図どおりか突き合わせます。
