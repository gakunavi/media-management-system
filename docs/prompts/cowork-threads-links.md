## 依頼

### 1. 週次生成で `article_link` を埋める（最優先）

生成時に、シートの `article_link` 列へ次の形式で入れてください。

```
https://collect.asset-support.co.jp/r/soken/{その行の id}
```

- `{その行の id}` は **同じ行の `id` 列の値**（例 `THR-612`）。
  シートを見たところ id は行の作成時点で採番されているので、生成時に確定します。
  ここが投稿IDと一致しないと、投稿単位の集計ができません（MMS は `THR-\d+` 以外を
  「サイト側の導線」として別扱いにします）。
- **UTM・`?from=`・LPのA/B振り分けは MMS が自動で付けます。書かないでください。**
  手で付けると二重になり、GA4 側の流入元集計が汚れます。
- 遷移先の使い分け:
  - `soken` … メディア（記事）へ送る ＝ ゴール①
  - `lp` … 診断LPへ送る
  - `line` … 公式LINEへ送る

### 3. 在庫（未投稿の draft/pending）にも入れる

キューに未投稿が約100本あります。**今後の生成分だけでなく、この在庫にも
`article_link` を入れてください**。入れないと、リンク付き投稿が実際に流れ始めるまで
1〜2週間かかり、その間ゴール①は未計測のままになります。

### 4. 集客コンテンツからのDMを記録するか決めたい

`dm-log.md` は代理店募集トラックのDMだけを記録しています。
そのため MMS は **集客コンテンツ（521投稿）から来たDMを「未計測」**として扱っています。

- 集客投稿からもDMが来ているなら、`dm-log.md` に載せる運用にしたいです
  （列順と判定4語「有効 / 有効候補 / 保留 / 無効」は変えないでください。MMSが依存しています）
- 来ていないなら「来ていない」と教えてください。MMS 側の表示を「実測ゼロ」に直します

## 変えないでほしいもの

- `dm-log.md` の列順と判定4語（`dm-log-import-daily` が毎日10:00に読んでいます）
- シートの `id` / `article_link` / `status` / `notes` の列名
- `notes` 先頭のフォーマット名（MMS が型の集計に使っています。
  `早口 | champion` のような実験マーカーは `|` の後ろに置けばそのまま扱えます）

## 入ったかどうかの確認方法

1. MMS の `/threads` →「ゴール」タブを開く
2. 「投稿はどのゴールを狙っているか」の **「狙い未設定」が減り「メディア送客」が増える**
3. リンクが1回でも踏まれると、①のカードが `—(未計測)` から実数に変わる
   （クリック率・1投稿あたりのクリック数も同時に出ます）

作業後に「何本の在庫に入れたか」「週次生成の何本に入れる設定にしたか」を返してください。
